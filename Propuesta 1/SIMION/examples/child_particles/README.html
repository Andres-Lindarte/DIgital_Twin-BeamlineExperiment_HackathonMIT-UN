SIMION Child Particles Example

Purpose/Description

This example demonstrates how to apply a user program to simulate the effect of a parent particle being replaced by multiple child particles during the trajectory calculation.

It is easy to transform one particle (the parent) into another particle (the child) just by modifying the ion_mass, ion_charge, and other reserved variables in an other_actions user program segment. Doing so can simulate the effect of ionization or secondary emission. However, if there are multiple child particles, then it is a little more complicated. There are various ways to solve this.

In SIMION 8.2 early access mode, a new particle API is provided to allow creating new particles during the Fly'm. This is illustrated in child.iob/child.lua. In SIMION 8.0/8.1, this API is not available, so some special tricks are required as shown in child80.iob/child80.lua.

SIMION 8.2/Early Access Method

The child.iob/child.lua example creates new particles by invoking the simion.experimental.add_particles function at any point during the Fly'm. See the documentation on the simion.experimental.add_particles function in the link above for details.

Older SIMION 8.0/8.1 Method

The approach taken in SIMION 8.0/8.1 is fairly general but still relatively simple. After the parent particle finishes flying, we prevent its termination, transform it into its next child, and continue flying. After that child finishes, we prevent its termination and transform it into the next child. This repeats until all the children of the parent are flown, after which the particle is allowed to really terminate. In this way, we use one particle in SIMION to trace the parent and all of its children.

See the child.lua file for details. Note: This program ASSUMES particles are flown individually ("Grouped" disabled).

A few alternative approaches could be preferable for your situation, and so they are mentioned here. One very simple method is to fly two identical ions and convert one to the first child and the other to the second child. That involves no additional user programming. In certain statistical studies, you might fly a random ion and then randomly make the decision of whether to convert it to the first or second child. That would simplify the user program, though if you're counting child particles (e.g. for transmission), you will need to double your counts.

Usage: child.iob

  • 1. View child.iob. (Click View on main screen and select the child.iob file.)
  • 2. Fly ions. (In Particles tab, click Fly'm.) Note: each parent particle is replaced by two child particles that go in different directions. Each child particle has a different color. Child particles then have further secondary emissions on hitting electrodes. You may try this with Grouped flying enabled or disabled (Grouped checkbox on Particles tab).

Usage: child80.iob

This example is somewhat similar. You must ensure Grouped is disabled though. (In Particles tab, ensure Grouped is unchecked.).

You can modify the is_debug adjustable variable (in the Variables tab) to turn debug logging on and off in the Log window. By default, debug logging and data recording are on, which you can use to confirm that the program is operating as expected.
