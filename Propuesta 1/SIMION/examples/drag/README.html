SIMION Stokes' Law Drag Example

Purpose

This example demonstrates the application of a Stokes' law [1] drag to ion motion. It uses an accel_adjust program segment.

Description

A simple three element fast adjust lens is provided for demonstration. This should serve as a simple example of how viscosity can impact ion motions.

Refer to the comments in drag.lua for details.

Files in this Demo:

  • README.html - Description of example
  • drag.iob - SIMION workbench
  • drag.fly - SIMION particle definitions associated with emittance.iob
  • drag.pa* - SIMION potential array of simple three-element lens
  • drag.con - SIMION contour definitions
  • drag.lua - SIMION user program associated with emittance.iob. This performing the tuning.
  • drag_mobility.iob/lua - Related example that calibrates the drag factor to a mobility constant.

Usage

How to run demo:

  • 1. View drag.iob. (Click View on main screen and select the drag.iob file.)
  • 2. If this is your first time using the example, you will be asked to refine (confirm that and press Refine).
  • 3. Set trajectory quality to zero. (Set TQual in Particles tab to 0.)
  • 4. Fly ions Grouped and in Rerun mode (Enable Grouped and Rerun options on Particles tab and then click Fly'm.) By default, there is zero dampening. Particles will refly continuously; so continue to next step.
  • 5. Increase the damping variable. (In the Variables tab, set linear_damping to, say, 3 and press Enter.) Note the effect of damping on ion trajectories.
  • 6. Now set middle electrode to zero volts to observe the effects of viscosity on diverging acceleration. (In PAs tab, click Fast Adjust Voltages, change electrode 2 to 0 V, and press Fast Adjust.)
  • 7. Be sure to check out PE views too! (PE button with XY, ZY, or XZ buttons pressed down.)

Note:

  • How beam refraction with damping active impacts ion spread!!
  • How divergence paints the walls with ions!
  • How convergence is impacted by viscosity (no focus point)!
  • Ions moving in viscous conditions follow streamlines. Use drag.con contour file to demonstrate. Viscosity is a nifty way to get streamlines.

More (beam repulsion effects):

  • 1. Set for Beam Repulsion at 1.0e-8 amps and trajectory quality = 0. (In Particles tab, select Repulsion as Beam and enter 1.0e-8 below it. Set TQual to 0.)
  • 2. Set left electrode at 1000V, middle = 1000V, right = 0V. (In PAs tab, click Fast Adjust Voltages, change voltages on three electrodes, and press Fast Adjust.)
  • 3. Fly ions Grouped in Rerun mode and damping zeroed. (In Particles tab, enable Grouped and Rerun options. In Variables tab, set linear_damping variable to 0 and press enter. Press down the Fly'm button.) Note ion trajectories.
  • 4. Try various damping factors. (In the Variables tab, change damping to 1 then 2 and so on.)

Note how viscosity really can boost repulsion effects!

drag_mobility example

The original drag.lua example above has the disadvantage that the Stokes' law drag factor is an empirical constant that might not be known in advance. Fortunately, however, given the drag factor in Stokes' law, we can derive a corresponding mobility coefficient (K = v / E), which is the ratio of the ion's terminal velocity v to a uniform electric field E it is submersed in, for a certain background gas composition, temperature and pressure. Mobility constants may be known from literature or experiment. We can furthermore reverse this process to estimate a drag factor from the mobility coefficient. The "drag_mobility.iob/lua" example does just that: it adapts the drag.lua example so that the drag coefficient is calibrated against a given mobility coefficient. For details on this calibration, see the section titled "Converting K mobility to damping constant" in the SDS documentation (examples\collision_sds\collision_sds_documentation.pdf). In fact, the code in drag_mobility.lua is incorporated into Example: SDS collision model to implement the mobility component of the combined mobility + diffusion effects in SDS. However, unlike the SDS example, the drag_mobility example omits diffusion effects; this simplifies the implementation but may be less realistic than SDS.

Stokes' law, which applies an additional transient force to ion motion, is not the only way to incorporate the mobility equation into the particle trajectory calculation. After all, the mobility equation, v = K*E, alone does not describe transient behavior like Stokes' law but is rather a steady state condition between an ion's time-averaged terminal velocity v and electric field E. This is only a steady state condition because the ion velocity v does not instantaneously track changes to electric field E according to the proportionality constant K. Not even in a perfect vacuum, where the Lorentz force law applies (F = m*dv2/dt2 = q*E), is this the case. Rather, it takes some time, sometimes referred to as the relaxation time (as in Example: FAIMS documentation). However, as the gas pressure is increased the terminal velocity v decreases and the time to reach terminal velocity decreases. At sufficiently high pressure, this time may be negligible small such that v = K*E may be assumed to hold for all time t. In that case, we can simply force the ion velocity vector according to the mobility equation at every time step. The `SDS_faims_mode = 1` in Example: FAIMS takes that approach. (FAIMS often operates at atmospheric pressure and the E field rapidly oscillates according to an RF waveform.)

Here's a few very basic sanity tests for this example (apart from the testing already done in the collision_sds example):

    Check #1:
      Set
        Zero initial ion KE, ion mass = 100, ion charge = 1 u.
        High background gas velocity (e.g. local_vx_mm_per_usec = 100)
        Zero fields (adjust all potentials to zero)
        STP conditions (local_pressure_torr=760, local_temperature_K=273.15)
      Record TOF, X position, and Vx at each time step.
      The particle should reach a terminal velocity equal to the background gas
      velocity in a short time.  The time required is pressure dependent--e.g.
      lowering pressure to 1 torr significantly increases this time.
      Increasing the field can alter the terminal velocity too because the field
      applies an additional force.

    Check #2:
      Set
        Zero initial ion KE.
        Zero background gas velocity (local_vx_mm_per_usec = 0). 
        STP conditions (local_pressure_torr=760, local_temperature_K=273.15)
        Default fields (fast adjust V1=1000, V2=1000, V3=0).
      Record ion velocity on every time-step.
      We observe the ion reach a terminal velocity of 4.8 mm/usec on the right side.
      This is consistent with the predication from the mobility equation:
      v = K*E = (2.0 * 10-4 m^2 V^-1 s^-1) * (~24 V/mm) = 4.8E-3 mm/usec.

    Check #3:
      The relaxation distance (see FAIMS documentation) is approximately
      lambda = (m/q)*E*K^2 = (100 u / 1e)*(~24 V/mm)*(2.0 * 10-4 m^2 V^-1 s^-1)
      = 1.0E-6 mm.  This is approximately the order of magnitude distance it takes
      for a particle starting at X=80 mm (right field) and zero KE to reach terminal velocity.
