SIMION Programmatically-Defined Magnetic Field Example (mfield_adjust segment)

Purpose

This example demonstrates how to create a magnetic field from an analytical equation defined in a user program (mfield_adjust segment). It also demonstrates overlapping a magnetic field defined as such over an electric field (defined by refining an electrostatic PA).

Background

There are a couple ways to generate a magnetic field in SIMION, as described in Supplementary Documentation: Magnets. In all cases, you first create a magnetic potential array, which is a potential array with the potential type set to magnetic. When you position an instance of this magnetic potential array on your workbench, the volume of the potential array instance is the volume in which particles will see the magnetic field. The magnetic field that SIMION applies in that volume is by default the gradient of the magnetic scalar potentials stored in the potential array. Alternately, if a Lua user program is attached to your workbench and that program contains an mfield_adjust segment, then SIMION will instead use the magnetic field defined programmatically by the mfield_adjust segment. In this example, the Lua user program (magnetic.lua) is very simple and defines only a uniform magnetic field whose vector components are adjustable variables, like this: 

simion.workbench_program()

adjustable bx = 15 -- Gauss
adjustable by = 0  -- Gauss
adjustable bz = 0  -- Gauss

function segment.mfield_adjust()
  ion_bfieldx_gu = bx
  ion_bfieldy_gu = by
  ion_bfieldz_gu = bz
end

The mfield_adjust segment: Any time SIMION needs to determine the magnetic field at some particle position (which can be at least once per time-step), SIMION will call the mfield_adjust segment and use the magnetic field vector whose components were stored by the mfield_adjust segment in the ion_bfieldx_gu, ion_bfieldy_gu, and ion_bfieldz_gu reserved variables. These variables are in units of Gauss. (Also, if the PA instance is rotated on the workbench, this vector is relative to the PA instance not the workbench.)

Use of empty magnetic PAs: If a magnetic field is defined via an mfield_adjust segment, then then any scalar magnetic potentials stored in the magnetic PA are ignored, so it is ok (and usually recommended) for that magnetic PA itself to be empty. The magnetic PA can also be 2D, even if the field represented by mfield_adjust is 3D, which conserves RAM. The mag.pa in this example is an empty 2D cylindrical magnetic PA.

Overlapping electric and magnetic PAs: It is not possible for a single PA to contain both electric and magnetic fields; PAs are either one or the other. However, electric and magnetic PA instances may overlap. If a particle is located at a position contained inside multiple PA instances, the particle will see the electric field from the highest priority electric PA instance containing that position (if any), and it will see the magnetic field from the highest priority magnetic PA instance containing that position (if any). The PA instance priority numbers are displayed in the PA Instances list on the PAs tab.

Usage

How to run this demo:

  • 1. View magnetic.iob. (Click View on main screen and select the magnetic.iob file.) If this is your first time using the example, you will be asked to refine (confirm that and press Refine).
  • 2. Fly ions. (Click Fly'm button.) Note in the XY view that the particles partly focus due to the electric field from the electrodes. Furthermore, in the ZY view you see that the particles all start in the Z=0 plane but get rotated around the X axis, which is an effect from the magnetic field. Actual values of electric and magnetic fields observed by the particles are recorded in the Log tab window. Therefore, we know particles are observing both electric and magnetic field affects.
  • 3. Turn off the magnetic field (by selecting the Variables tab and changing to zero the "bx" variable, which represents the x component of the magnetic field in units of Gauss). Fly the particles again. Note that the beam now focuses very well and there is no rotation as seen before.
  • 4. Turn off the electric field and redefine a magnetic field with components 3 Gauss in the X and Y directions. (In PAs tab, select tune.pa0 (the electrostatic array), click Fast Adjust and set all electrodes to 0V. In the Variables tab, set "bx" and "by" both to 3.) Fly the particles again. Note that particles now have a purely helical trajectory, as expected from a uniform magnetic field not parallel to the particle velocity vector. This can be seen best using a 2D zoom in the ZY view.

Other Directions

Defining the magnetic field with an arbitrary equation: The above example merely defines a uniform magnetic field. You can customize the magnetic.lua program to define the magnetic field according to any arbitrary equation. For example, edit the magnetic.lua file in a text editor (click "User Program..." on the Particles tab) and change the program to this (merely for demonstration): 

simion.workbench_program()

function segment.mfield_adjust()
  ion_bfieldx_gu = ion_px_mm^2 / 100
  ion_bfieldy_gu = ion_px_mm^2 / 100 
  ion_bfieldz_gu = 0
end

If you disable the electric field and fly this, you'll then see that the oscillations get smaller and smaller as the particle moves along the X direction due to the increasing magnetic field in the X direction.

Biot-Savart calculation: The magnetic field may be much more complex. SIMION includes a Biot-Savart magnetic field solver, which derives the magnetic field produced from an abitrary configuration of wire current, and this can be inserted into the mfield_adjust segment as well. See Solenoid Example and Helmholtz Coil Example for demonstrations of this.

Plotting fields: Magnetic fields defined via mfield_adjust, unlike those derived from the gradient of scalar magnetic potentials stored in the PA, are not normally visible to SIMION's PE and contouring views. These magnetic fields can still be observed though with SIMION Data Recording (as seen in the Log tab of this example). Even though PE and contour views do not see magnetic fields defined via mfield_adjust, SIMION 8.1 provides another way to plot this field via user programming, as seen in Contour Example, as well as Solenoid Example and Helmholtz Coil Example which also use this approach.

Earth's magnetic field varies between 0.3 to 0.6 Gauss (see Wikipedia:Earth's magnetic field), which is important to keep in mind, and the approach in this example may be used to simulate this effect. If the effect is significant, you may need to apply magnetic shielding in your instrument, such as with mu metals (see Supplementary Documentation: Magnets) or a helmholtz coil, to counteract this effect.
