SIMION Secondary Emissions Example

Purpose

This files in this directory provide an example of having user programs model secondary emission effects. When a particle hits a surface, it deflects in some specified way, optionally changing mass, charge, direction, and energy.

Description

This is a more advanced example with a fairly complex user program. There a dozen or so parameters that you may need to adjust depending on what secondary emission characteristics you desired. See the comments in the Lua programs, particularly secondarylib.lua for a detailed description of the secondary emission capabilities and the adjustable parameters.

About the Model

Files in this demo:

  • README.html - Description of example.
  • secondary.iob - SIMION workbench.
  • secondary.gem - SIMION GEM file used to create secondary.pa#
  • secondary.pa* - SIMION potential array.
  • secondary.lua - SIMION user program associated with secondary.iob.
  • secondarylib.lua - Implementation of secondary emission effects. Used by secondary.lua.
  • history.lua - Utilty module for recording data along trajectories. Used by secondarylib.lua.
  • secondary.fly2 - default particles
  • secondary-test.fly - particle for accuracy test
  • secondary2.fly2 - other particles for experimenting with

Usage

How to run the demo:

  • 1. View secondary.iob. (Click View on main screen and select the secondary.iob file.)
  • 2. If this is your first time using the example, you will be asked to refine (confirm that and press Refine).
  • 3. Fly particles. (Click Fly'm.)

Experimenting with directions.

  • 4. Fly particles. (Click Fly'm.) Note how particles hit electrode surfaces and reflect. By default, specular reflection is used, where angle of incidence equals angle of reflection.
  • 5. Switch to normal reflection mode and refly. (In Variables tab, change secondary_direction_mode to 2 for normal reflection. Click Fly'm.) Note the particles now reflect at an angle perpendicular to the surface regardless of their incident angle.
  • 6. Switch to Lambertian reflectange mode and refly. (In Variables tab, change secondary_direction_mode to 3 for Lambertian reflection. Click Fly'm.) Note that particles now reflect in randomized directions. In Lambertian reflectance, the intensity observed at an angle θ relative to the surface normal is proportional to cos(θ).
  • 7. Switch back to specular reflection mode. (In Variables tab, change secondary_direction_mode to 1 for specular reflection.

Experimenting with emission probability and repeats.

  • 8. Set secondary emission probability to 0.5 and refly. (In Variables tab, change secondary_probability to 0.5.) Note: only about half of the particles reflect now.
  • 9. Set secondary emission probaility back to 1. (In Variables tab, change secondary_probability to 1.)
  • 10. Set the maximum number of repeated secondary emissions to infinite and refly. (In Variables tab, change secondary_max_times from 1 to 1e30 (approximately infinite). Click Fly'm.) Note: particles reflect many times, filling the volume until the particles eventually escape through the gap between the two electrodes.
  • 11. Set the maximum number of repeated secondary emissions back to 1. (In Variables tab, change secondary_max_times to 1.)

Experimenting with mass/charge of secondaries.

  • 12. Experiment with changing the mass and charge of secondaries. (In Variables tab, set secondary_mass to, say, 100 amu, and secondary_charge to, say, -1. Note: the default large values 1e30 tell SIMION to use the same mass and charge as the incident particles. Click Fly'm.) Note: how secondaries have a charge and so are affected by the fields and follow curved paths.
  • 13. Set mass and charge of secondaries to original settings. (In Variables tab, set secondary_mass and secondary_charge both to 1e30.)

Experimenting with energies.

  • 14. Set secondary energy mode to fractional with fraction of 0.5. Refly and view energies in log. (In Variables tab, ensure secondary_energy_mode is 2 for fraction of incidence energy. Set secondary_energy_value to 0.5 so that secondaries are half the energy of incident particles. Click Fly'm. Select Log tab and click "W" to open log window.) Note: in the log window, incident hits ("hit_event") shows ke=100 eV while secondary particles ("secondary_event") shows ke=50 eV.
  • 15. Set secondary energy mode to constant 40 eV. Refly and view energies in log. (In Variables tab, set secondary_energy_mode to 1 for constant energy. Set secondary_energy_value to 40. Click Fly'm. View log window.) Note: In log, secondary particles ("secondary_event") now shows ke=40eV.
  • 16. Set secondary energy mode to randomized Maxwell-Boltzmann with k*T of 5 eV. Refly and view energies in Log. (In Variables tab, set secondary_energy_mode to 3 for Maxwell-Boltzmann energy distribution. Set secondary_energy_value to 5. Click Fly'm. View log window.) Note: secondary particles now show ke values randomly distributed with energies roughly the same order of magnitude a secondary_energy_value.
  • 17. Set secondary energy mode back to fractional with fraction 1. (In Variables tab, set secondary_energy_mode to 2 for fractional energy. Set secondary_energy_value to 1.)

Accuracy and implementation topics.

  • 18. Load secondary-test.fly2. (In Particles tab, click Define.... Click Load and select secondary-test.fly2. Click OK.)
  • 19. Set max secondary repeats to 30 and fly. (In Variables tab, set secondary_max_times to 30. Click Fly'm.) Note that particle retraces back and forth along rectangular path many times according to geometry, but eventually numerical error accumulates and particle deviates off this path contrary to geometry.
  • 20. Change secondary offset to 10 grid units and refly. (In Variables tab, set secondary_offset_gu to 10. Click Fly'm.) Notice that the trajectories near the curved surface are about the same but actual trajectory flying to the surface is avoided. This was mainly to show you what this parameter does, as normally, you don't set it this high.
  • 21. Change secondary offset to 0 grid units and refly. (In Variables tab, set secondary_offset_gu to 0. Click Fly'm.) This effectively disables the secondary offset correction. Notice that the trajectories deviate more quickly from the theoretical rectangle. This demonstrates how this correction can be useful.
  • 22. Set secondary offset back to 2 grid units. (In Variables tab, set secondary_offset_gu to 2.)
  • 23. Enable surface normal estimation from field (rather than analytic definition) and refly. (In Variables tab, set secondary_normal_mode to 1 (estimation from fields. Click Fly'm.). Notice the increased errors. It is much more accurate to define surface normals analytically. You might experiment with secondary_offset_gu here too.
  • 24. Switch surface normal calculation back to analytic. (In Variables tab, set secondary_normal_mode to 2 (analytical definition.)

A Few General Notes

When Number of Particles Varies over Secondary Emission

Generally speaking, in most simulations, with more than a handful of particles, we are only simulating the traces of a statistically representative sample of the population of particles that physically exist. These may be referred to as "super particles" (super electrons). Now, if the number of physical particles varies over time or distance (e.g. due to secondary emission), we have two choices, which are alternate ways of looking at the problem:

  • (1) The number of physical particles that each super particle represents (i.e. the weight or current assigned to the super particle) could be kept fixed over the trace, while allowing the number of super particles to vary as needed during the simulation.
  • (2) Alternately, we may keep the number of super particles fixed over (or at least non-increasing) during the simulation but vary their weights or currents as needed. This can be done by adjusting each particle's charge weighting factor (CWF), which is the ion_cwf variable, in a user program.

If you have a high current amplification in a series of dynodes in an electron multiplier, one complication that can occur with the former way of thinking (#1) is that you start out tracing a small number of super particles, but at the latter dynodes you are then tracing an enormous number, perhaps orders of magnitude more. The ray tracing calculation time depends on the number of super particles, which therefore increases exponentially over the trace with the former approach. Perhaps another limitation is that the number of physical particles (or current) that may be represented at any one time would be artificially forced to be a discrete multiple of the number physical particles represented by a single super particle. This might not be as much of an issue though if the number of particles or the total amplification is low though.

If you do want to do #1, one way to kind-of simulate it is by starting M number of sets of identical particles with the same position and velocity and each representing a fractional 1/M of a particle. These particles will follow overlapping trajectories and together appear to be a single particle until the first secondary emission event, in which case they will randomly diverge and multiply by M in current (so that each now represents a full particle). At least they will do what you ask for the first emission event. In theory, SIMION users programs allow writing custom code that could force this to behave more generally however you want (see Example(child_particles).
