SIMION Spectrum Acquisition Example

Purpose

This example illustrates how to acquire a spectrum by performing multiple reruns, each at a different electrode voltage, and counting the number of particles that are accepted. Multiple scans can be done, and the cumulative spectrum can be displayed interactively in your graphical plotting program (e.g. Excel--see Example(plot)). For background, see Doc(spectrum).

Model Description - Parallel Plate Analyzer (PPA)

A simple ideal 2D parallel plane analyzer (PPA) is used for this example. The bottom plate is ground and the top plate voltage is scanned. Electrons enter though a slit in the bottom plate and some or all subsquently exit through another acceptance slit in the bottom plate. For a pass energy E for charge q0, the plate voltage difference must be V = k E, where k=2 (1/q0) d0/x0, and d0 and x0 are the distances between the plates and slits respectively. A plot of particle acceptance count v.s. pass energy is plotted.

Usage

How to run demo:

  • 1. View spectrum.iob. (Click View on main screen and select the spectrum.iob file.) If this is your first time using the example, you will be asked to build/refine (confirm that and press Refine).
  • 2. Fly ions. (Click Fly'm.)
  • 3. A cumulative scan will display in your graphical plotting program (e.g. Excel) at the end of each step. You may disable the graphical plotting by changing the "use_plot" adjustable variable (on the Variables tab) from 1 to 0, and a histogram will still display in the Log window. There are also other adjustable variables seen in spectrum.lua. For example, setting plot_frequency=2 will update the graph only at the end of each scan, and plot_type=0 will generate a scatter plot.
  • 4. You may quickly change the exit slit width to 10 mm by setting the slit_width adjustable variable to 10 (on the Variables tab) and then clicking Fly'm. Notice the effect of slit width on both peak width and intensity.

Notes

Slit width adjusting: You may quickly change the exit slit width by adjusting the slit_width adjustable variable (on the Variables tab) and then clicking Fly'm or entering regenerate() in the SIMION command bar. This will automatically rebuild the PA instance from the GEM file and refine it. The regenerate function is implemented by spectrum.lua. regenerate(true) will force a regeneration even if the dimensions have not changed.

Slit effects and hit detection: In this example, some particles pass through the exit slit but still miss the detector. You may try to improve this by changing the detector size, position, or voltage. Other particles may begin to pass through the exit slit but splat against the sides of the slit, thereby decreasing resolution. Ideally, you want the width of the plate near the slit to be designed as small as possible to avoid this, though not so small that electrons tunnel through the plate. In SIMION, "solid" electrodes, which splat particles, are at a minimum one grid unit in width, while zero grid unit width electrode surfaces allow particles to pass through (unless you force a splat via an ion_splat=-1 in a user program other_actions segment). Therefore, you may need to reduce the grid cell size, at least in the y direction (see Anistropically scaled grid cells), in order to represent your desired electrode plate thickness accurately. A quick way to determine, or disregard, whether the slit depth is affecting results is to change the hit detection code in spectrum.lua to count hits against the slit sides as well. You could alternately change the exit slit to a solid electrode in the PA and change the detection code to count particles hitting certain areas of the bottom plate, as if the detector were part of the bottom plate and its shape determined entirely by the hit detection code. This approach could even model a slit smaller than one grid unit and would neglect detector miss effects as well. A test plane (e.g. testplanelib.lua in Example: Test Plane) may also be useful at times for the detection code if the particles you want to count aren't actually hitting an electrode but rather passing through some region of space, like this: 

local TP = simion.import'testplanelib.lua'   -- load test plane library

-- Define test plane by point vector and normal vector and use callback function
local test1 = TP(100,0,0, 0,1,0, function()
  if ion_px_mm >= 99.5 and ion_px_mm <= 100.5 then  -- accept subset of points in plane
    count = count + 1  -- HIT COUNTING
    mark()  -- just for visualization
    print('counting', ion_number, ion_time_of_flight)
  end
end)

-- Call test plane segments from your regular SIMION segments.
function segment.tstep_adjust()
  test1.tstep_adjust()
end
function segment.other_actions()
  test1.other_actions()
end

Scan speed: The present model treats the electrode voltages as being constant during each step in the scan or, more specifically, during the time-of-flight of each particle traced. Here, TOF is a fraction of a microsecond, though in IMS it could be milliseconds. Whether this makes any difference depends on your scan speed relative to your TOF. If this is a concern, you could modify this example to continually increase the electrode voltage during each run (i.e. by incorporating ion_time_of_flight in the fast_adjust segment). You could furthermore do one SIMION run per scan (rather than one run per step in the scan), give the particles a range of time-of-birth (TOB) values in the particle definitions, and measure hit counts over intervals of time to generate a histogram. Such an approach may be more general and realistic.
