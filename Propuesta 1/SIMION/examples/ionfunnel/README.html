SIMION Ion Funnel Example

Purpose

This example models an "ion funnel," which is a device consisting of a series of ring electrodes stacked in a funnel shape with RF and DC voltages applied to focus ions in the presence of a background gas (see papers below for background). The model here is a good example of the techniques of applying a collision model to an RF system. It also demonstrates efficiently modeling an RF system with a large number of electrodes by using a much lower number of "fast scalable" electrodes when voltages on many of those electrodes are constrained by linear relationships over all time. (Mathematically, we determine a small number of Laplace solutions and apply the principle of superposition to quickly obtain the Laplace solution at any time t.)

Files in this demo:

  • README.html - Description of example
  • funnel.iob - SIMION workbench
  • funnel.fly2 - SIMION particle definitions associated with funnel.iob
  • funnel.gem - Geometry definition that may be used to generate funnel.pa#
  • funnel.pa# - SIMION definition potential array of funnel
  • funnel2.* - Equivalent SIMION example showing more efficient use of fast scalable electrodes (two electrode solution arrays)
  • funnel2.pa+ - Defines fast scalable electrodes for funnel2.pa#.
  • funnel3.* - Similar to funnel2.iob but uses additional electrode solution arrays to allow adjustable variables to be adjusted during the Fly'm (without editing the .pa+ file and re-refining the array).

Description

Modeling

Ions are generated randomly in energy, cone angle, radial offset, and phase using a FLY2 file (funnel.fly2), which is viewable and editable from "Particles|Define..." on the SIMION View screen.

There are 16 electrodes in the funnel, plus a DC electrode at the end. The electrodes are 0.5 mm thick and separated 0.5 mm, similar to in [1+2].

The voltages on the first 16 funnel rings at time t is

Vk = (_DC_offset_1 + dcgradient * (k-1)) + _RF_amplitude * (-1)k-1 sin(t * ω + θ)

for k=1..16 and certain constants _DC_offset_1, dcgradient, ω, θ, and _RF_amplitude. RF and DC voltages are applied to the funnel electrodes with a user workbench program, funnel.lua (or funnel2.lua/funnel3.lua). More detailed discussion of the model and its parameters is contained in the comments in the funnel.lua file.

The program incorporates an ion-neutral collision model (SIMION HS1-collision model - collision_hs1.lua), which is duplicated in this folder. (The collision cross section used is m/z dependent.)

Notes on the Voltages in the Second Example (funnel2.iob)

There is a second example (funnel2.iob) included that behaves identically to the first example (funnel.iob) but more efficiently uses electrodes to achieve higher speed and memory efficiency. funnel2.iob models the system using only two "fast scalable" electrodes (adj_elect01 and adj_elect02). Multiple fast scalable electrodes are newly supported in SIMION version 8.0.3 (for background, see "Additional Fast Scalable Electrodes in SIMION 8.0.3"). These electrodes are defined in the funnel2.pa+ ASCII text file.

The first scalable electrode (adj_elect01) represents the system that consists of all voltages Vk such that sin(t * ω + θ) = 1:

Ak = (_DC_offset_1 + dcgradient * (k-1)) + _RF_amplitude * (-1)(k-1)

The second scalable electrode (adj_elect02) represents the system that consists of all voltages Vk such that sin(t * ω + θ) = -1. In other words, it is 180 degrees out of phase from the first voltages:

Bk = (_DC_offset_1 + dcgradient * (k-1)) - _RF_amplitude * (-1)(k-1)

The value of adj_elect01 is defined as the maximum of Ak, which is A1 here. The value of adj_elect02 is defined as the maximum of Bk, which is B2 here. Note that B2 is lower (by dcgradient) than A1.

We will show now that the voltages Vk on the 16 rings can be obtained by taking a linear combination of these two solutions.

The voltages are controlled by the following statements in the user program:

sf = 0.5 * (1 + sin(ion_time_of_flight * omega + theta))
adj_elect01 = sf * max1
adj_elect02 = (1 - sf) * max2

where the constant sf oscillates between 0 and 1 over time.

That causes the voltage on each ring k to be the linear combination of the two solutions:

sf * Ak + (1 - sf) * Bk
= sf * [(_DC_offset_1 + dcgradient * (k-1)) + _RF_amplitude * (-1)(k-1)] +
  (1 - sf) * [(_DC_offset_1 + dcgradient * (k-1)) - _RF_amplitude * (-1)(k-1)]
= (_DC_offset_1 + dcgradient * (k-1)) + _RF_amplitude * (-1)(k-1) * (2 * sf - 1)
= (_DC_offset_1 + dcgradient * (k-1)) + _RF_amplitude * (-1)k-1 * sin(t * ω + θ)
= Vk

which is shown to be identical to the Vk voltage used the first system, funnel.iob.

That's the full derivation, but intuitively this is all fairly simple (though maybe more-so if you have taken a course in linear algebra). The voltages on the electrodes at any time t can be expressed as linear combinations of two different sets of voltages on those electrodes.

It's convenient that there are times (sf=0 and sf=1) at which adj_elect01 or adj_elect02 is zero. In this case, the voltages on the rings are due only to adj_elect01 or adj_elect02 and are at their extrema. At sf=1, we have adj_elect01 = max1 and adj_elect02 = 0. At sf=0, we have adj_elect01 = 0 and adj_elect02 = max2. max1 and max2 refer to the maximum voltages on rings 1 and 2 respectively, which are directly measurable quantities.

WARNING: if you change DC and RF voltages in funnel2.iob, you will need to both change the adjustable variables in funnel2.lua and change the corresponding variables in funnel2.pa+ (and then re-refine funnel2.pa#). To avoid this inconvenience, see funnel3.iob. As an exception, the above equations do actually allow the RF magnitude to be adjustment if we merely rescale the factor (2 * sf - 1). "funnel2.lua" takes that into account.

Notes on the Voltages in the Third Example (funnel3.iob)

funnel3.iob (with corresponding funnel3.pa+ and funnel3.lua) is similar to funnel2.iob but uses additional electrode solution arrays to allow adjustable variables (_RF_amplitude, _DC_offset_1, _DC_offset_16, and _DC_offset_17) to be adjusted during the Fly'm (without editing the .pa+ file and re-refining the array).

Usage

How to run funnel demo:

  • 1. View funnel.iob. (Click View on main screen and select the funnel.iob file.)
  • 2. If this is your first time using the example, you will be asked to refine (confirm that and press Refine).
  • 3. Select trajectory quality = 0. (In Particles tab, set TQual to 0.)
  • 4. Fly ions. (Click Fly'm.) Note how ions are focused toward the vertex of the funnel.
  • 5. Experiment on the effects of tuning parameters and ion initial conditions. (Variables tab and Particles|Define...)

How to run the funnel2 and funnel3 demos:

  • The funnel2.iob and funnel3.iob examples can be operating simularly.
  • You can even verify that both systems are identical by turning off all sources of randomization in each: (1) convert the .FLY2 file to an .ION file (In the Particles tab, click Define... and select "Individually") and set the _pressure_pa variable to zero to disable randomized background gas collision (In the Variables tab, set _pressure_pa to 0). Compare trajectories.
