SIMION PA Instance Jumping (Non-ideal Grid) Example

Purpose / Description

This example demonstrates the simulation of particle scattering by non-ideal grids using an "PA instance jumping" trick. The method allows accurate simulation of a non-ideal grid inside a larger system. This example is based on Example: nonideal_grid\particle_jump, which uses an analogous "particle jumping" trick to achieve a similar result. The difference is that here we jump the PA instance rather than jump the particle to model to fine non-ideal grid region. Jumping the PA instance can be simpler, more natural, and require less coding.

This is an advanced SIMION simulation. You should be familiar with the use of user programs before attempting this simulation. Please examine comments included within the program file (ref70b.lua).

Compatibility Note

This example uses SIMION 8.1 features.

Files in this example

  • README.html - description of this example
  • ref70b.iob - workbench
  • *.pa* - potential arrays
  • source.gem - GEM file used to generate source.pa
  • blank.gem - GEM file used to generate blank.pa
  • mgrid70b.gem - GEM file used to generate mgrid70b.pa#
  • ref70b.rec - data recording file
  • ref70b.fly - ion definitions
  • ref70b.lua - workbench user program (performs PA instance jumping trick)

Usage

  • 1. Load the .IOB file. (Click View/Load Workbench, select ref70b.iob.) Select Yes if asked to build/refine the PAs.
  • Note: the GEM file gives a model of a 70 lpi grid. If you use a different mm/gu scaling factor (default 0.004) or a different grid size (default 136 x 136 x 272), you will need to adjust the GEM file, DUX/DUY variables in the user program, and the mm/gu scaling factor on the PA instance in the workbench view.
  • 2. Fly particles. (Click Fly'm.) You may wish to set the display to ZY view and set the display quality (Qual) to 8. By appropriate 2D and 3D zooms you can observe the very small nonideal grid PA instances moving during the fly'm. Note the small effects on the spacing between particle trajectories due to the grid. Look at the ref70b.lua file to see how the PA jumping is coded.
