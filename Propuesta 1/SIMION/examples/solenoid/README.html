SIMION Solenoid Magnetic Field Example

Purpose

This is an example of inserting an air-core solenoid magnetic field into a workbench. The magnetic field is calculated via Biot-Savart as a series of wire loops. The field is loaded into the workbench via a small user program.

Files in this demo:

  • README.html - Description of example
  • solenoid.iob - SIMION workbench
  • solenoid.fly2 - SIMION particle definitions associated with solenoid.iob
  • mag.pa - Blank magnetic PA. Used to contain solenoid magnetic field. Used in solenoid.iob.
  • solenoid.lua - SIMION workbench user program used to apply solenoid magnetic field into solenoid.iob workbench.
  • toroid.iob/lua - Additional toroidal solenoid example
  • solenoid_write.lua - batch mode program to write solenoid field to CSV text file.

Usage

How to run the solenoid example:

  • 1. View solenoid.iob. (Click View on main screen and select the solenoid.iob file.)
  • 2. If this is your first time using the example, you will be asked to refine (confirm that and press Refine).
  • 3. Fly ions. (Click Fly'm.) Notice that the ions are focused through the cylindrical region (which is inside the solenoid). When the particle is in a region greater than 9 Gauss, the trajectory is drawn green (for display purposes).

See the solenoid.lua source code for more details. Also, try the toroid.iob example.

Field Caching Option

There is an option to improve trajectory calculation performance by caching Biot-Savart magnetic field calculations. This is option is contained in the solenoid.lua code.

A caution: If you don't know what this section of code does, it's safer to just delete it. This code is intended to increase calculation speed at the expense of some in accuracy. In some cases it may even reduce speed too. Therefore, it's safest not to use this option unless you really need, know it benefits v.s. costs are significant, and apply it properly.

The problem: Biot-Savart calculations are sometimes computationally intensive. At every point in which the magnetic field needs to be determined, a big Biot-Savart integration is done. This may be a lot of points: in this example, the Biot-Savart code is invoked from an mfield_adjust segment, which is called for every particle position at every time-step. Luckily, that integration is pretty optimized in SIMION, but tons of wire turns or time steps can still bog it down.

The basic solution: Speed may be improved, however, by not recalculating Biot-Savart on every point required but rather only calculating the magnetic field on the vertex points of a grid cell surrounding each local set of required points and using (fast) interpolation between those vertex points to determine the field of all required points inside that grid cell. Furthermore, you only need to do this for grid cells that particles actually pass through.

The implementation: The CachedField object handles all of this behind the scenes. The CachedField object provides the same external interface as the Biot-Savart field object, but internally it works differently. When you query the field at a point with the CachedField, it first tries to interpolate a value between the nearest grid points if that field has already been calculated (cached) at those grid points; otherwise, it invokes the provided function (here, the Biot-Savart field calculator) to determine the fields at those vertex points and caches those values for future use. In this code, the current field object is replaced with a cached field object that wraps the original field object.

Implications: This option is intended to reduce runtimes at some expense in accuracy. In some cases, particularly inside cells where fields are queried multiple times, the cached field will improve run-time; in other cases, it may actually increase run-times. Larger grid cells (as well as 2D rather than 3D cells) improve caching but also introduce greater loss in accuracy from interpolation.

solenoid_write.lua

Click Run Lua Program on the SIMION main screen and select the solenoid_write.lua file. The solenoid field will be written to a text file (out.csv) containing solenoid field data. That file can be examined in Excel.
