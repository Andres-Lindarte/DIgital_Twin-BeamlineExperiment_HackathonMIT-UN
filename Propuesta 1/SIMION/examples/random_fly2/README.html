SIMION Example - Passing Adjustable Variables to a FLY2 File

Purpose

This example demonstrates how to reload a FLY2 file from a workbench user program and pass adjustable variables into the FLY2 file. The example is based on Example: Random.

Files in this demo:

  • README.html - Description of example
  • random.iob - SIMION workbench
  • random.fly2 - Particle definitions. This was edited manually with a text editor to insert variables passed from the user program. DO NOT resave it from the GUI because this will overwrite the content.
  • random.lua - SIMION workbench user program. This defines adjustable variables and passes those into the FLY2 file.
  • random.rec - SIMOIN data recording definitions (optional)
  • random.pa# - example PA file (simple geometry of anode, cathode, and focus electrodes).
  • particlelib.lua - utility library to assist in reloading a FLY2 file and passing variables to it. You can largely ignore the contents of this file. The random.lua uses it.

Description

The "random.lua" file defines adjustable variables that controls the about of angular and KE randomization in the beam. The "random.fly2" file defines starting conditions for a conical beam. This file was edited directly with a text editor to introduce variables into it: 

-- Read in any variables.
local var = _G.var or {}
local percent_energy_variation = var.percent_energy_variation or 0
local cone_half_angle = var.cone_half_angle or 0

The user program reloads the FLY2 file and passes adjustable variables into it like this: 

adjustable percent_energy_variation = 50
adjustable cone_half_angle =  90

function segment.flym()
  local PL = simion.import 'particlelib.lua'
  -- Regenerate particle definitions in case FE cathode properties changed.
  PL.reload_fly2('random.fly2', {
    -- variables to pass to FLY2 file.
    percent_energy_variation=percent_energy_variation,
    cone_half_angle=cone_half_angle,
  })
  run()
end

Caveat: Do not re-save the FLY2 file from the SIMION GUI. This will overwrite the special Lua code inserted into the FLY2 file. Only edit the FLY2 file with a text editor (e.g. Windows Notepad).

SIMION 8.2

In SIMION 8.2 (or early access mode in 8.1), it is even more convenient using the simion.experimental.add_particles function: 

simion.workbench_program()
adjustable percent_energy_variation = 50
adjustable cone_half_angle =  90
adjustable radius = 0

function segment.initialize_run()
  simion.early_access(8.2)
  local F = simion.fly2
  simion.experimental.add_particles {
    F.particles {
      F.standard_beam {
        n = 20, tob = 0, mass = 100, charge = 1, cwf = 1, color = 2,
        ke = F.uniform_distribution {
          min = 0.1 * (1 + percent_energy_variation/100),
          max = 0.1 * (1 - percent_energy_variation/100)
        },
        direction = F.cone_direction_distribution {
          axis = F.vector(1, 0, 0),
          half_angle = cone_half_angle, fill = true
        },
        position = F.circle_distribution {
          center = F.vector(2, 5, 0),
          normal = F.vector(1, 0, 0),
          radius = radius, fill = false
        }
      }
    }
  }
end

Usage

How to run demo:

  • 1. View random.iob. (Click View on main screen and select the random.iob file.) If this is your first time using the example, you will be asked to refine (confirm that and press Refine).
  • 2. Fly ions. (In Particles tab, Click Fly'm button.)
  • 3. On the Variables tab, change cone_half_angle from 90 to 2 (degrees) and click Fly'm again. Notice how the angle of the beam decreased. Explore how other adjustments to these variables affect the output to the graphical window and the Log window.
