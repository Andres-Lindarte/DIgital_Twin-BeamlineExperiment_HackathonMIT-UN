SIMION Waveform Example

Purpose

This examples demonstrates how to apply a segmented waveform (e.g. square, triangular, or complex segmented-line pulse) to an electrode.

Files in this demo:

  • README.html - Description of example
  • plates.iob - SIMION workbench
  • plates.fly2 - SIMION particle definitions associated with IOB.
  • plates.rec - SIMION data recording definitions associated with IOB.
  • plates.pa* - SIMION potential array of two parallel plates in 2D.
  • plates.lua - SIMION workbench user program that applies the waveform to adjustable electrodes.
  • waveformlib.lua - library of reusable SIMION Lua code to apply waveforms to adjustable electrodes. This is used by plates.lua and can be copied/pasted into your own simulation (often without modification).

Modeling

The model consists of two parallel plate electrodes whose potentials can be independently adjusted (.PA# file). A single particle flies between the plates, initially parallel to the plates (+X direction). Voltage pulses are applied to the plate electrodes, which deflect the particle in the +-Y direction.

Usage

How to run demo:

  • 1. View plates.iob. (Click View on main screen and select the plates.iob file.)
  • 2. If this is your first time using the example, you will be asked to refine (confirm that and press Refine).
  • 3. Fly ions in PE View. (Click on the PE button and click Fly'm.) The particles are flown and you'll see the electrodes potentials (quickly) changing over the time the particle flies. Output is sent to the Log window, and a plot of the waveform is also generated in Excel. (Note: if you don't have Excel, you may remove the line "WAVE.plot_waveform()" from plates.lua to disable Excel plotting.) Switch to XY View and open the Log window (click the XY button and click the Log tab.) Notice that the pulse on the first plate causes the particle to accelerate and move in the +Y direction and decellerate to zero velocity in the Y direction. Notice that the subsequent pulse on the second plate causes the particle to accelerate and move in the -Y direction and decellerate to zero velocity in the Y direction. After the second pulse, the particle has returned to the X axis (note that the Log window displays a Y value very close to zero when the particle terminates -- this is expected from theory).
  • 4. To further explore, view the source code of the user program plate.lua. (In the Particles tab, click User Program...) Note how this user program defines the two waveforms. Also, for convenience, it enables periodic PE updates in the display and plots the waveform as well. This code is fairly simple. Most of the complexity is hidden behind the library "waveformlib.lua", which in general you won't need to touch (and it is fairly complex). You may reuse waveformlib.lua in your own simulations: just copy the waveformlib.lua into your own project folder and attach a workbench user program similar to plates.lua to your own workbench (but adapted to define the particular waveform you would like to use). waveformlib.lua is quite general and can be used to apply arbitrary segmented waveforms to an arbitrary number of electrodes.

See Also

  • Example: Buncher Lens defines a much simpler waveform (single half-pulse). It does not use waveformlib.lua but rather defines everything itself. This example may be more useful if you can't or don't want to use waveformlib.lua (e.g. if you want to do something else).
  • Example: RF Demo, Example: RF Quadrupole Filter, and Example: RF Octupole (as well as others) define sinusoidal RF waveforms.
