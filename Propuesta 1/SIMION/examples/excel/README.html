SIMION Excel Examples

Purpose

These examples demonstrate invoking Excel from a SIMION user program. This is done via the Excel COM interface. Examples included do things like creating a scatterplot or surface plot, which can be useful for analyzing simulation results.

Please note that if you want your code to also be usable with other plotting programs besides Excel (e.g. gnuplot), then it is recommended to instead use the generic plotting library available in Example(plot). This generic library provides a standard interface for working with both the Excel and gnuplot libraries.

Description

These examples assume you have Microsoft Excel (possibly requiring a recent version) installed.

Files in this demo:

  • README.html - Description of example
  • scatterplot_test.lua - Lua batch mode program that generates a scatter plot in Excel. The scatterplot contains random points for demonstration. This mainly just shows how to call Excel from Lua.
  • tof_vs_ion.iob/lua - example plotting TOF v.s. ion number.
  • append_chart.iob - example of appending to existing worksheet/chart [8.1]
  • phaseplot_old.lua - old version of examples\plot\phaseplot.lua (phase plotting) that does not use excellib.lua.
  • excellib.lua - library that facilitates simplifies plotting 2D data with Excel. This is also used by examples\plot\plotlib.lua.

Usage

The scatterplot_test.lua example can be run as follows:

  • In SIMION, click "Run Lua Program" and select "scatterplot_test.lua". A scatterplot is created in Excel and displayed on the screen. (A more advanced method is to do this from the command-line: From this project directory type "..\..\simion.exe --nogui lua scatterplot_test.lua".)

tof_vs_ion.iob and be run by loading it and clicking Fly'm.

Further Notes

For more information on controlling Excel from Lua, see the Microsoft Excel Object Model. You can find this in the Microsoft Excel Help, in Contents, under the topic "Microsoft Excel Help\Programming Information\Microsoft Excel Visual Basic Reference\Microsoft Excel Object Model" (might be located elsewhere depending on your version of Excel). There is also an online version. The examples in the Microsoft Documentation are largely written in the VisualBasic lanuage, so some translation is required to similarly do this under Lua. See the SIMION Excel examples and the LuaCOM reference for an idea of how to perform this translation.

Additional Examples

This will load a comma-separated-value (CSV) text file into Excel. It does not use COM but rather just os.execute. You may need to change the path if Excel is installed at some other location.

os.execute[[start "" "c:\Program Files\Microsoft Office\Office10\EXCEL.EXE" one.csv]]

This creates a row of values, creates other cells with formluas defining the sum and average of those values, and reads the calculated values in those formula calls.

local excel = luacom.CreateObject("Excel.Application")
excel.Visible = true
local wb = excel.Workbooks:Add()
local ws = wb.Worksheets(1)
for row=1,100 do
   ws.Cells(row,1).Value2 = math.random()
end
ws.Cells(1,2).Formula = "=SUM(A1:A100)"
ws.Cells(2,2).Formula = "=AVERAGE(A1:A100)"

local sum     = ws.Cells(1,2).Value2
local average = ws.Cells(2,2).Value2

print(string.format("sum=%f,mean=%f", sum, average))

This shows how to update a range of cells.

local excel = luacom.CreateObject("Excel.Application")
excel.Visible = true
local wb = excel.Workbooks:Add()
local ws = wb.Worksheets(1)

-- Update entire range of cells.
ws:Range("A1:B5").Value2 = 10
ws:Range("A1:A5").Font.ColorIndex = 5  -- blue
ws:Range("A3:A5").Font.Bold = 1

To prevent Excel prompting if you want to save on closing, set the "Saved" property to "true" on the workbook:

local excel = luacom.CreateObject("Excel.Application")
excel.Visible = true
local wb = excel.Workbooks:Add()
...
wb.Saved = true

Here's how to set the x range on chart axes (using excellib.lua):

local EXCEL = simion.import 'excellib.lua'
local t = { {5,10}, {6,12}, {7,11} }
local plot = EXCEL.plot(t)
plot.chart:Axes(1).MinimumScale = -5
plot.chart:Axes(1).MaximumScale = 10

Excel Errors

It's best to avoid trying to use Excel while LuaCOM is writing to Excel. 

COM error:(src\library\tLuaCOM.cpp,391):Call was rejected by callee.
stack traceback: [C]: ? excellib.lua:?: in function 'update_data'
The above can occur when Excel is busy for some reason and LuaCOM is trying to call it. For example, you may be changing a chart font in Excel while SIMION is updating the chart. 
COM error:(src\library\tLuaCOM.cpp,391):Unknown error
stack traceback: [C]: ? excellib.lua:?: in function 'update_data'
This can occur when switching from Chart1 to Sheet1 in Excel when SIMION is updating the chart.

Excel Versions

This code will work with most Excel versions, although different Excel versions may have minor differences. However, Excel "Starter Edition" doesn't support COM automation, so the besy you can do is transfer data between SIMION and Excel via .csv or tab-delimited text files. If you have multiple versions of Excel installed, see Office automation when multiple versions of Office are installed .
