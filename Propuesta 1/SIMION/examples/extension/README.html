SIMION Extension Modules Example

Purpose

This example demonstrates calling external programs or C/C++ code from SIMION via the Lua interface.

Description

As outlined in Calling External Programs, there are various ways that SIMION can call external code or programs:

  • Calling external programs via os.execute or io.popen Lua commands. The program must support a command-line interface. There is the overhead of starting up the program, so this is typically not suitable if you need to call the code, say, a million times per second.
  • Calling external programs via COM (LuaCOM). The program must support a COM interface (e.g. like Excel). The call may be in-process or out-of-process depending on the object but in either case is relatively efficient for doing that.
  • Writing some code in a language like C, compiling it to a DLL, and loading that into Lua. This method can be very efficient but requires some coding.

These methods are all described below.

Calling an external program

The Lua "os.execute" function can be used to run an external program. The Lua "io.open" function is similar but captures any any data written to standard output as a string. API details on these functions are in the Lua Reference Manual os and io libraries.

For example, this will open a certain file in Windows Notepad: 

os.execute 'notepad.exe "c:/Program Files/SIMION-8.1/README.html" '

This will start Windows Notepad but not block SIMION: 

os.execute 'start notepad.exe "c:/Program Files/SIMION-8.1/README.html" '

Be careful about backslashes "\" and forward slashes "/". For strings surrounded by single or double quotes, backslashes must be escaped with an additional backslash. You can often avoid this by just using forward slashes or quote strings with "[[ ]]", which don't need or recognize backslash escape sequences. The following three commands are all valid: 

os.execute 'notepad.exe "c:\\Program Files\\SIMION-8.1\\README.html" '
os.execute 'notepad.exe "c:/Program Files/SIMION-8.1/README.html" '
os.execute [[notepad.exe "c:\Program Files\SIMION-8.1\README.html" ]]

In some cases, the Windows program or shell does not recognize forward slashes and you need to instead use backslashes. Be careful also about spaces in program arguments; for example, the space in "Program Files" requires the full path to be quoted. According to Windows conventions, some of these quoting rules are a little obscure. Some resort to writing a batch file (.bat) to disk and then just invoking that batch file without arguments.

See also Example: gnuplot, which invokes the gnuplot plotting program via these two functions.

The command line interface may also be used to execute scripts in other languages, like Python/Perl/Matlab: 

os.execute 'python myprogram.pl my.txt'

COM

Some programs like Excel can be automated via COM. Lua can access COM objects via LuaCOM. See Example: Excel for details.

Calling code in an external DLL

The following example demonstrates calling C/C++ code in an external DLL. Lua loads the DLL in-process and executes functions inside it.

Files in this Demo:

  • README.html - Description of example
  • simple.c - C code for extension module.
  • simple.cpp - C++ code for extension module (alternative to simple.c)
  • x32/simple.dll - precompiled 32-bit DLL (provided for convenience)
  • x64/simple.dll - precompiled 64-bit DLL (provided for convenience)
  • test.lua - test program to call simple.dll
  • Makefile - Makefile for building the module.
  • CMakeLists.txt - CMake build (alternative to Makefile if you prefer cmake)

Usage - Calling code in an external DLL

This is an advanced example that assumes familiarity with building DLLs in C (or C++) and using the Lua C API. For details, the book Programming in Lua and the resources on www.lua.org are recommended.

How to use the C demo:

  • 1. Note: a precompiled simple.dll is provided for convenience so that you can run the example without compiling. 32-bit and 64-bit DLLs are in the x32 and x64 folders respectively. To use the precompiled simple.dll, skip to step 9; otherwise, proceed to step 2.
  • 2. Make a temporary copy of the "extension" example. For example, in Windows explorer, copy "c:\Program Files\SIMION-8.1\examples\extension" to "c:\temp\extension".
  • 3. Make sure a C/C++ compiler is installed. Examples include
    • Cygwin 1.7 with MinGW GCC - This is relatively quick way to get working on Windows via a Linux-like interface. Install Cygwin. During installation, make sure you install one of more of these packages: "mingw64-i686-gcc-core" (32-bit native C compiler), "mingw64-i686-gcc-g++" (32-bit native C++ compiler), "mingw64-x86_64-gcc-core" (64-bit native C compiler), and "mingw64-x86_64-gcc-g++" (64-bit native C++ compiler). We also recommend the "make" package.
    • MinGW/MSYS - This is somewhat similar to the Cygwin 1.7 install (which includes MinGW). See http://mingw-w64.sourceforge.net/ or www.mingw.org/ .
    • Visual Studio or the free Visual Studio Express (C++) 2008/2010. Note: As of 2010, the Express Edition doesn't support 64-bit compilation without special configuration.
    • Windows SDK - This contains 32-bit and 64-bit command-line compiler tools without the Visual Studio IDE.
    • Yet other compilers are also available. Take your pick.
  • 4. Open up a command prompt to access your compiler.
    • If using MinGW/MSYS (GCC), start the MSYS shell.
    • If using Cygwin 1.7 (With MinGW GCC), start Cygwin (shortcut to Cygwin.bat).
    • In using Visual Studio, select the Windows Start Menu "Microsoft Visual Studio > Visual Studio Tools > Visual Studio Command Prompt" (for 32-bit) or "Microsoft Visual Studio > Visual Studio Tools > Visual Studio x64 Win64 Command Prompt" (for 64-bit).
    • If using the Windows SDK, select the Windows Start Menu "Microsoft Windows SDK v7.0 > CMD shell". You can use the command "setenv /x86" to switch to 32-bit mode or "setenv /x64" to switch to 64-bit mode.
    • If using some IDE, open that instead.
  • 5. Navigate to the temporary folder you created.
    • If using a command prompt enter "cd c:/temp/extension" .
    • If using an IDE, create a project and add simple.c to it.
  • 6. For some compilers, you may need to build an "import library" for the lua51.dll. This need only be done once and can be reused for all projects.
    • If you are using the GCC C compiler in MinGW (or MinGW from Cygwin 1.7), you do not need an import library.
    • If you are using the Microsoft C/C++ 2008/2010 compiler or the Windows SDK 6a, there is an import library already built for you. The library located at c:\Program Files\SIMION-8.1\lib\cpp\lua51.lib is either a 32-bit or 64-bit version depending on which your operating system is. On 64-bit operating systems, a 32-bit version is located at c:\Program Files\SIMION-8.1\x32\lib\cpp\lua51.lib . Alternately, you may build the import library yourself from the lua51.def file (as may be necessary if using a different version of the compiler) by running the command "nmake lua51-msvc32" (for 32-bit) or "nmake lua51-msvc64" (for 64-bit) inside that directory. You will need to edit the Makefile to reference the new .lib file.
  • 7. Compile simple.c to simple.dll.
    • If you are using the GCC C compiler in MinGW (including from Cygwin 1.7), run the command "make simple-mingw32" (for 32-bit DLLs) or "make simple-mingw64" (for 64-bit DLLs).
    • If you are using the Microsoft C compiler, run the command "nmake simple-msvc".
  • 8. Copy simple.dll into the x32 subdirectory (if it's 32-bit) or x64 subdirectory (if it's 64-bit). The "test.lua" example will search for the DLL in these locations. Alternately, if you want simple.dll to be globally visible to all SIMION projects without modifying package.cpath, you may copy simple.dll into the "c:\Program Files\SIMION-8.1\lib\lua\" directory (which requires administrative priviledges). (If using the 32-bit version of SIMION on a 64-bit OS, the 32-bit DLL should instead go into "c:\Program Files\SIMION-8.1\x32\lib\lua\".)
  • 9. Test it. Click "Run Lua Program" on the SIMION main screen and select the "test.lua" example to run. test.lua adds either the x32 or x64 subdirectory (as appropriate) to the package.cpath search path, loads the "simple" module, and calls a function inside it. This example prints "1+2=3".

How to use the C++ demo:

  • The C++ example is almost identical to the C one, but the source and build differ slightly. Compile simple.cpp (instead of simple.c) to simple.dll (this can be set via the SOURCE variable in the Makefile).

Error Messages

"error loading module XXXX from file '...XXXX.dll': %1 is not a valid Win32 application." This typically means, for example, that you are trying to load a 32-bit DLL into a 64-bit process (EXE) or vice-versa. Windows only allows 64-bit DLL's to be loaded into 64-bit bit processes and 32-bit DLL's to be loaded into 32-bit processes. So, either recompile the DLL as 64-bit or use the 32-bit DLL in the 32-bit SIMION (note: on 64-bit operating systems, SIMION installs a 32-bit version of the EXE in the "x32" subdirectory of the SIMION 8.1 program folder).
