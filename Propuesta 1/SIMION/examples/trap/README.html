SIMION Ion Trap Example

Purpose

The files in this directory provide an example of having user programs model an ion trap.

Description

About the User Programs

There are three demos - each with its own user program file and FLY files:

  • group.lua - user program to demonstrate motions of groups of ions in the trap - ions form patterns and shells due to charge repulsion and gas cooling (simulated by Stoke's law)
  • inject.lua - user program to demonstrate injection of ions external to the trap. Includes the effects of collision gas (simulated by simple mean-free-path collisional cooling)
  • tickle.lua - user program to demonstrate ejection of ions using resonant end cap voltage techniques. Includes the effects of collision gas (simulated by mean free path collisional cooling)
  • group_hs1.lua - similar to group.lua but demonstrating incorporation of the HS1 collision model.

All programs share the traputil.lua code. The programs also use a variety of collision models (collision_stokes.lua, collision_simple.lua, and collision_hs1.lua).

About the Model

The potential array trap.pa# represents a practical or non-ideal ion trap approximating an ideal hyperbolic field using mostly circular (rather than hyperbolic) electrodes. The ring electrode radius r0 is ~ 4.1 mm and the distance between edge cap electrodes 2*z0 is ~ 5.8 mm. This follows the rule for ideal hyperbolic (and non-stretched) ion trap that r02 = 2*z02.

By default, the end cap electrodes are grounded at 0 V and the ring electrode is oscillated with sinusoidal potential U + V * sin(θ + t * Ω) for time t and certain constants U, V, θ, and Ω.

Assuming an ideal system with particles of mass/charge, m/e, the stability of the system in dimensions z and r is determined by the Mathieu constants:

az = -2ar = -8eU/(mr02Ω2)

qz = -2qr = 4eV/(mr02Ω2)

The user programs in this example allow the user to conveniently control the voltages through the Mathieu parameters az and qz. Typically, we set az = 0, in which case an ideal trap would be stable in the approximate region 0 < qz < 0.908. Rearranging qz gives the mass/charge ratio at the instability limit:

m/e = 4V/(qzr02Ω2)

Files in this demo:

  • README.html - Description of example.
  • group.iob - SIMION workbench - illustrates ion grouping in trap
  • inject.iob - SIMION workbench - illustrates ion injection into trap
  • tickle.iob - SIMION workbench - illustrates tickle voltage to extract ions
  • group_hs1.iob - SIMION workbench - illustrates using HS1 collision model instead
  • trap.pa* - SIMION potential array for ion trap.
  • traputil.lua - Utility functions and variables shared by the three programs (inject.lua, group.lua, and tickle.lua).
  • collision_hs1.lua - HS1 hard sphere collision model
  • collision_simple.lua - simple elastic collision model (simplification of HS1)
  • collision_stokes.lua - Stokes' law (viscount drag) collision model
  • inject.fly - SIMION particle definitions for inject.lua.
  • inject1.fly - SIMION particle definitions. This is an alternative to inject.fly but with multiple masses.
  • group.fly - SIMION particle definitions for group.lua.
  • group1.fly - SIMION particle definitions. This is an alternative to group.fly but with multiple masses.
  • tickle.fly - SIMION particle definitions for tickle.lua.
  • group_hs1.fly - SIMION particle definitions for group_hs1.lua.

Usage

How to run "group" demo:

  • 1. View group.iob. (Click View on main screen and select the group.iob file.)
  • 2. If this is your first time using the example, you will be asked to refine (confirm that and press Refine).
  • 3. Select Grouped, Factor Repulsion of 1, trajectory quality = 0 or -1, and Dots with max speed. (In Particles tab, enable Grouped. Set Repulsion to Factor and enter 1 in the text box below that. Set TQual to 0 or -1. Enable Dots and move nearby slider to very top for max speed.)
  • 4. Switch to PE view and set view quality to zero and GStep to 8 (faster). (Press down PE button. In Display Qual, enter 0. In PE/Contours tab, set Gstep to 8.) Watch the PE surface undulate!
  • 5. Exit PE view to view ions fly with default settings. (Unpress PE button.)
  • 6. Switch to PE view. (Press down PE button) Watch the PE surface undulate!
  • 7. When ions converge look at end view and notice pattern (interesting). (To view, first unpress PE button. In ZY view, drag rectangle around ions and click Z2D to zoom in. Alternately use a 3D zoom.) Note: If you instead use a 3D zoomed view of the inside of the trap press down Hidden button on Display tab so ions will fly faster. Switch trajectory quality to -3 or lower (more stable results).
  • 8. Fly group1.fly ions under the same conditions. (Ensure Fly'm button is not pressed. In Particles tab, click Define..., click Load, select group1.fly file, and click OK. Click Fly'm to restart flight.) Notice the ions form shells, with heaviest on the outside (blue = 300, green = 200, red = 100 amu). This is stratification [1-3].
  • 9. Set _linear_damping to zero and see what happens. (In Variables tab, set _linear_damping to 0 and press enter.)
  • 10. Set Factor Repulsion to 1.0e4 and notice size of ion cloud. (In Particles tab, in the text box below Repulsion, type 1.0e4 and press enter.)
  • 11. Modify adjustable variables while ions are in flight. (Use Variables tab.) You can change the operating point of the trap dynamically (QZ and AZ), potentials on the endcaps, damping, and etc. Hours of great fun!
  • 12. Add your own program modifications to perform the tests you desire!

Note: the group_hs1.iob example uses the HS1 collision model instead.

Note: SIMION has successfully duplicated experiments with cooled ions both in ion pattern and size of the ion pattern e.g. (Observation of a Phase Transition of Stored Laser_cooled Ions, F. Diedrich and all, Physical Review Letters Volume 59 Number 26 1987 APS)

How to run "inject" demo:

  • 1. View inject.iob. (Click View on main screen and select the inject.iob file.)
  • 2. If this is your first time using the example, you will be asked to refine (confirm that and press Refine).
  • 3. Select Grouped, no repulsion, trajectory quality = 0, and dots with max speed. (In Particles tab, enable Grouped. Set Repulsion to None. Set TQual to 0. Enable Dots and move nearby slider to very top for max speed.)
  • 4. Fly ions. (Click Fly'm button.) 100 ions are randomly injected into trap.
  • 5. Re-fly with different mean free path values and see what happens. (In Variables tab, change _mean_free_path to say 0.1 or 100. Toggle Fly'm button on again to refly.) Note: Lower values (e.g. 0.1) cause more ions to stay in the trap.
  • 6. Fly inject1.fly ions under the same conditions. (In Particles tab, click Define..., click Load, select inject1.fly file, and click OK.) Notice the heavier ions are more efficiently trapped than lighter (red) 100 amu with QZ at 0.8). Rerun with QZ at 0.4. (In Variables tab, change _qz_tune to 0.4 and press enter.) This gives better 100 amu (red) capture.
  • 7. Rerun above with mean free path at 4.0. (In Variables tab, change _mean_free_path to 4 and press enter.) This may be more realistic.
  • 8. Modify adjustable variables while ions are in flight. (Use Variables tab.) You can change the operating point of the trap dynamically (QZ and AZ), potentials on the endcaps, MFP, etc.
  • 9. Add your own program modifications to perform the tests you desire!

How to run "tickle" demo:

  • 1. View tickle.iob. (Click View on main screen and select the tickle.iob file.)
  • 2. If this is your first time using the example, you will be asked to refine (confirm that and press Refine).
  • 3. Select Grouped, no repulsion, trajectory quality = 0, and Dots with max speed. (In Particles tab, enable Grouped. Set Repulsion to None. Set TQual to 0. Enable Dots and move nearby slider to very top for max speed.)
  • 4. Fly ions. (Click Fly'm button.)
  • 5. Set tickle voltage to 2 volts and Fly ions. (In Variables tab set _tickle_voltage to 2 and press enter. Press down Fly'm button if not already down.) Note: the ions are blue when tickle pushes them to right. The ions are red when tickle pushes them to left. Increase tickle voltage to blow ions out of trap.
  • 6. Rerun but at a different tickle frequency. (Adjust _tickle_frequency in Variables tab.) Note phasing effects of being widely off frequency.
  • 7. Put some DC voltage on the right end-cap and see what effect it has. (Adjust _right_cap_voltage in Variables tab.) You can also change the operating point of the trap dynamically (QZ and AZ), potentials on the endcaps, damping, and etc.
  • 8. Add your own program modifications to perform the tests you desire!
