SIMION Time of Flight (TOF) Example

Purpose

The files in this directory provide an example of a simple reflectron time-of-flight (TOF).

Files in this demo:

  • README.html - Description of example
  • tof.iob - SIMION workbench
  • tof.fly - SIMION particle definitions associated with einzel.iob
  • source.pa* - SIMION PA of TOF source (where ions originate)
  • mirror.pa* - SIMION PA of TOF reflectron
  • detect.pa* - SIMION PA of TOF detector (where ions hit)
  • tof.rec - SIMION data recording definition

Usage

How to run the TOF demo:

  • 1. View tof.iob. (Click View on main screen and select the tof.iob file.)
  • 2. If this is your first time using the example, you may be asked to refine (confirm that and press Refine).
  • 3. Set trajectory quality to 2. (In Particles tab, set TQual to 2.)
  • 4. Fly ions Grouped as Dots. (In Particles tab, enable Groups and Dots. Click Fly'm button.) Note: ions of the same mass are the same color.
  • 5. Note the time focusion effect of the reflectron.
  • 6. Switch to YZ view and cutaway left half of all optical elements via a 3D zoom. (In ZY View, drag rectangle around slightly more than left half of screen and press +Z3D.)
  • 7. Now refly the ions and view from 3D isometric. (Press down 3D Iso button and click Fly'm button.)
  • 8. Turn on data recording and load the tof.rec file. (In Particles tab, enable Record data and click Data Recording... button. Click Load, select tof.rec file, and click OK.)
  • 9. Fly ions again. (Click Fly'm button) Note that their times of impact are recorded. (Click Log tab to view recorded results.)
  • 10. Experiment.

More Comments

In this example, mirror.pa is the reflectron, where most of the interesting field deflection is. "source.pa0" defines an acceleration field to get particles initially moving toward the reflecton. "detect.pa" does not have any fields in it, but it does have an electrode used to stop particles exiting the reflectron. You can observe that in the PA edges where particles enter and exit each of these PA's, there is an ideal grid with potential of zero (Doc(grid) for more notes about ideal grids). Therefore, fields between the PA's would be zero, which is what SIMION assumes when you break the system into multiple PA's like this. In practice, there would be some fringing fields outside the PA's if there grids were not exactly ideal, if you removed the grids or if the if the other edges are not shield with with ground plates, so this is somewhat of an approximation to break this into multiple PAs rather than use one potential array.
