SIMION Buncher Example

Purpose/Description

The files in this directory provide an example of having user programs model an ion buncher. The problem is to slow a packet of ions so that they converge in time. The trick is to have the ions decelerate in a linear field and then (when the ions are in the middle of it) chop the field to zero. Thus the leading ions will be going slower than the trailing ions and they will bunch together at some point downstream. This should serve as a simple example of how buncher design problems might be approached.

Files in this demo:

  • README.html - Description of example
  • buncher.iob - SIMION workbench
  • buncher.fly - SIMION particle definitions associated with buncher.iob
  • buncher.pa* - SIMION potential array of simple three-element buncher
  • target.pa - SIMION potential array of hit target
  • buncher.lua - SIMION user program implementing voltage control of buncher. Associated with buncher.iob.

Usage

How to run demo:

  • 1. View buncher.iob. (Click View on main screen and select the buncher.iob file.)
  • 2. If this is your first time using the example, you will be asked to refine (confirm that and press Refine).
  • 3. Set trajectory quality to 3 for good splat numbers. (In Particles tab, TQual should be 3. This is the default.)
  • 4. Fly ions Grouped and as Dots. (In Particles tab, enable Grouped and Dot. Click Fly'm.) Note: Ions are marked when field is chopped. Also ion color is changed from red to blue at transition. You may switch off Dots to view trajectories as lines.
  • 5. Use PE view to get a better understanding of how bunching works. (Press down XY button and then PE button.) Note: In PE view, the potential energy surface changes dynamically under program control, showing how the buncher works.
