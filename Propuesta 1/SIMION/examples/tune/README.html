SIMION Voltage Optimization Examples

Purpose

These are simple examples of doing electrode voltage optimization. The simulation is automatically run many times using different values of voltages for each run.

Description

The files in this directory provide an example of having user programs tune a lens. It serves as a moderately complex control problem and should be studied for tricks of the trade. This should serve as a simple example of how tuning problems might be approached.

A simple three element fast adjust lens is provided for tuning. The middle electrode is adjusted via a user program.

Files in this demo:

  • README.html - Description of example
  • tune.iob - SIMION workbench, traditional binary search voltage optimization
    • tune.fly - SIMION particle definitions associated with tune.iob
    • tune.pa* - SIMION potential array of simple three-element lens
    • tune.lua - SIMION user program associated with tune.iob. This performing the tuning.
  • tune_binaryoptlib.iob - SIMION workbench, binary search voltage optimization implemented as a separate library (easier to use)
  • tune_simplex.iob - SIMION workbench simplex optimizer.
  • tune_series.iob - SIMION workbench, iterating through a series of voltages A and B
  • tune_random.iob - SIMION workbench, iterates through a random sequences of voltages A and B. Optionally writes to Excel.

Usage

How to run demo (tune.iob):

  • 1. View tune.iob. (Click View on main screen and select the tune.iob file.)
  • 2. If this is your first time using the example, you will be asked to refine (confirm that and press Refine).
  • 3. Set trajectory quality to 2 (need good splat numbers). (In Particles tab, set TQual to 2.)
  • 4. Fly ions Grouped. (In Particles tab, enable Grouped and click Fly'm.) Tune results can be seen in the Log tab.
  • 5. Note: You can use the Variables tab to change focus goal during flights.

More (tuning in PE view):

  • When you tune in PE view the user programs automatically update the PE surface view to show surface currently being tested!

More (tuning with beam repulsion effects):

  • 1. Set for Beam Repulsion at 1.0e-7 amps and trajectory quality = 0. (In Particles tab, set Repulsion to Beam and enter 1.0e-7 in the text box below that. Set TQual to 0.)
  • 2. Fly ions Grouped. (In Particles tab, enable Grouped. Click Fly'm.)

Other examples (.iob files) can be used in a similar manner. See their associated .lua files for details on implementation.

Additional Notes

Implementation Comments

Voltages are changed between runs by fast adjusting, which is very rapid. This is achieved via a fast_adjust or init_p_values user program segment.

Applications

These examples were primarily designed for the purposes of optimizing electrode voltages. However, the basic concept may also be used for optimizing other conditions, such as starting particle conditions (e.g. position). Optimizing geometries is also a common need but is a little more involved because the potential arrays need rebuilt and re-refined on each iteration (see link to geometry optimization examples at bottom).

The Basic Approach

The basic goal of optimization is to find an input value x that minimizes the value y = f(x), where f is some function that we can evaluate but might not otherwise know much about. In the context of SIMION, we typically have that x is an electrode voltage (or a vector representing voltages of multiple electrodes), y is some property of the particle beam (e.g. beam radius or focal point), and f the simulation system. f is evaluated by doing a SIMION run. We can evaluate f many times, by inputting various values of x and measuring y each time.

Now, we could exhaustively evaluate f at all points over the domain of x with some precision. This could require a huge number of evaluations. For example, to find a global minimum of f over the domain x=[0,1] with four decimal point precision, this will generally require evaluating f at points x=0, 0.0001, 0.0002, ... 1.0000 (i.e. 10000 points). If, however, we know or suspect that f behaves in a certain way, we can do the search more intelligently. For example, if we know or suspect that f has a local minimum in the domain x=[0,1] and that f always increases when moving to the right or left of the minimum, then we can do a directed search: if f was previously evaluated at xi and xi+1, then the next point xi+2 is chosen toward the direction of xi if f(xi) < f(xi+1), or toward the direction of f(xi+1) otherwise. (This is similar to the children's "higher/lower" number guessing game.)

Automating reruns

This example performs a series of trajectory calculation runs from a "flym" segment, which is a new segment type in SIMION 8.1.0.40. The flym segment will initiate a run each time it calls run(). This call is typically preceeded by code to initialize the parameters of the run and followed by code to store the results of that run. See "Workbench Program Extensions in SIMION 8.1" (workbench_program_extensions) in the supplemental documentation (Help menu) for details. The following example illustrates the basic concept of doing three runs at three different voltages (100 V, 200 V, and 300 V): 

function segment.workbench_program()

local v0

function segment.flym()
  for i = 1, 3 do
    v0 = 100 * i
    run()
  end
end

function segment.fast_adjust()
  adj_elect01 = v0
end

Optimization Libraries

If you need to optimize a variable or two in your simulation, it's often simplest to use one of the existing optimization libraries rather than write your own code. A few libraries are provided: the simplex optimizer (simionx.SimplexOptimizer) or the binary search optimizer (binaryoptlib.lua). These are each demonstrated in the tune_simplex.iob and tune_binaryoptlib.iob examples. Either works well for optimizing a single variables. The simplex optimizer is more general and is required for optimizing multiple variables at a time.

The above optimizers only find local minima; they do not find global minima. To find global maxima, use tune_series.iob or tune_random.iob. tune_series.iob iterates through all voltages in linear sequence. tune_random.iob runs through a random sequence of voltages.
