SIMION Programmatic Test Plane Recording Example

Purpose

The files in this directory provide an example of defining and recording data on additional test planes via a user program. This can be more flexible that the "Crossing Plane" option in SIMION Data Recording. Here you can define multiple test planes, even test planes non-orthogonal to the axes.

Description

See the comments in einzel.lua and testplanelib.lua for additional details on this example. testplanelib.lua is designed to be reusable so that you can copy and use that file in your own simulations.

Files in this demo:

  • README.html - Description of example
  • einzel.iob - SIMION workbench
  • einzel.fly - SIMION particle definitions associated with einzel.iob
  • einzel.pa* - SIMION potential array of simple three-element lens
  • einzel.lua - SIMION workbench user program that applies test plane code (testplanelib.lua)
  • testplanelib.lua - Utility library (used by einzel.lua) for defining and recording data on additional test planes.

Usage

How to run demo:

  • 1. View einzel.iob. (Click View on main screen and select the einzel.iob file.) You will not be asked to refine it since this example comes pre-refined.
  • 2. If this is your first time using the example, you will be asked to refine (confirm that and press Refine).
  • 3. Fly ions. (Click Fly'm.) Notice the dots where the particles touch the three test planes. See also the Log window recording of data on those test planes. View the user program (einzel.lua) to see how the test planes are defined. Take a look at the comments in testplanelib.lua too.
