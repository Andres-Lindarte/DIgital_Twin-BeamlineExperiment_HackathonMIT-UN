SIMION Advanced Contour/Field Plotting Example

Purpose

SIMION's PE/Contours tab provides a number of built-in functions for visualizing fields and contour lines. However, there may be special cases where you want more flexibility for field plotting capabilities beyond these. User programming tricks can offer a solution. The examples here demonstrate the following:

  • Plotting field lines or field vectors. These are curves or line vectors parallel to the field. (Note: The usual "Potentials" and "Gradients" options under Contours in the PE tab do exactly not do this. "Potentials" plots equipotential lines, which are perpendicular to field lines. Equipotential lines and field lines (as well as PE surfaces) do convey much of the same information, but you may prefer the field lines/vectors display. "Gradients" is a plot of the magnitude (a scalar value) of the field, discarding directional information and so does not provide this either.)
  • Exporting paths of contour and field lines and data along those lines. The contour and field lines in these examples are obtained by forcing particles of your choosing to follow equipotential lines or field lines, and the trace of these particles form the lines in the display. A convenient side-effect of this is that you can now use SIMION's regular Data Recording options (or user programming other_actions segments for even greater flexibility) to record data about your field at each position a particle traverses (i.e. on every time-step).
  • Plotting field lines for fields defined in a user program (efield_adjust or mfield_adjust segments). The PE/Contours features only plot fields defined in potential array files, not fields defined programmatically such as from user programs (including magnetic fields computed by wire/solenoid currents using the simionx.MField library as demonstrated in Example: solenoid). The approach taken here does in fact identically handle both fields defined in potential arrays and fields defined programmatically, so it is particularly useful for the latter.
  • Plotting other field types, like pressure, temperature, velocity and density arrays as used in gas flow definitions (like in the SDS and SDS-FAIMS models).

About the Models - contourlib81.lua (SIMION 8.1)

The included library contourlib81.lua library will plot field vectors on the SIMION 8.1 View screen for any arbitrary given function. As a simple example, load the mag.pa PA file, enter the View screen, and then type this into the SIMION command bar: 

dofile'contourlib81.lua'.plot(function(x,y,z) return x,y^2,0 end)

The given vector function will be plotted over the entire workbench. See the comments in the contourlib81.lua source code for details on various options that may be passed to the plot function.

To clear the lines, use the Particles tab > Trajectory Display > Del button.

Usage - SIMION 8.1 examples

How to run the example:

  • Load the solenoid81.iob and click Fly'm. The magnetic field vector for the solenoid will be plotted. You may zoom in (+Z2D) to see lines things better. See the solenoid81.lua file for details.

About the Models - contourlib.lua (SIMION 8.0)

SIMION 8.0, unlike SIMION 8.1, doesn't have a function to directly plot lines on the View screen. However, one trick that has been used it force particles to follow the paths of the lines you want to draw. This approach is described below an is implemented in contourlib.lua.

The examples below perform contouring by using a separate Lua library file (contourlib.lua) that implements all the contouring logic. Normally, you can simply use contourlib.lua without understanding or modifying its contents. Simply attach a Lua workbench user program to your workbench, and from that workbench program import the contourlib.lua file as done in the below examples. Alternately, you may even use contourlib.lua directly as your workbench user program (by renaming it so that the base of the filename matches your workbench IOB file).

Once the contouring library is attached, the only thing else you need to do is define particles that the contouring library will use to trace the potential or field lines. If you define a particle with mass 1000, 1001, 1002, or 1003 u, the contouring library will take over that particle and use it to plot a potential or field line. The type of line drawn depends on the chosen mass as given below:

  • 1000 u - electrostatic equipotential contour
  • 1001 u - electrostatic field line
  • 1002 u - magnetic equipotential contour
  • 1003 u - magnetic field line
  • (other) - particle not used for contouring

Furthermore, a line can be traced in two directions (forward or back). Which one is traced is controlled by whether the particle charge is specified as positive or negative. You may want to start two particles (positive and negative) from the same point to obtain a full field line extending in both directions.

Usage - SIMION 8.0 examples

How to run the example:

  • 1. Plotting field lines.... View drag.iob and fly ions. (Click View on main screen and select the drag.iob file. If this is the first time running this example you may be asked to refine the array; if so, refine it as instructed. Click the Fly'm button.) The particles (blue) follow the field lines. Note that the field lines are, as expected, perpendicular to the contour lines (red) plotted using the normal options in the PE/Contours tab.
  • 2. Load drag-vector.fly2 and Fly ions. (In the Particles tab, click Define..., then click Load... and select the drag-vector.fly2 file.) Note that the particles are now defined as starting along the X and Y directions in arithmetic sequences. The particles will start at all points on a 2D grid covering the entire volume. (Click Fly'm. Ignore warning about some particles being created inside electrodes.) Observe the particle trajectories. The field lines now cover the entire system.
  • 3. Plotting field vectors... It may help visualization now if we make the field lines shorter and proportional to field intensity. Set the contour_min_gu and contour_max_gu adjustable variables to 0.5 and 4 respectively. (Do this in the Variables tab.) Enable Grouped flying and Fly ions. (In the Particles tab, ensure that Grouped is checked and click Fly'm.) The Fly'm will fail with error if you do not have Grouped checked since the code depends on this. Note the field vectors display over the entire volume with direction perpendicular to equipotential lines (as before) and length proportional to field intensity (you may want to switch to PE View to confirm this). Note that a small non-zero value of contour_min_gu was selected to prevent field lines becoming nearly invisible in small field regions.
  • 4. Plotting equipotential lines and recording data along those lines... For the sake of completion, let's now plot equipotential contour lines using the programming approach, even though we are already displaying equipotential lines using the normal PE/Contour tab features. (Load drag-equipotential.fly2.) (In the Particles tab, click Define..., click Load..., and select drag-equipotential.fly2.) Observe how the particle starting points are defined. They are equidistant along the x-axis. Also, for each of those points, particles of positive (+1) and (-1) charge are defined. This causes the contour plotting routine to plot the equipotential line in both forward and reverse directions starting at the given point. (Set adjustable variables contour_max_gu to 0 and contour_field to 0.) (Do this in the Variables tab.) contour_field = 0 results in a equipotential plot rather than and a field lines/vector plot. Fly ions now. (Click Fly'm.) Notice how the equipotential lines generated by the programming routine (blue) do indeed follow the equipotential routines generated using SIMION's normal PE/Contours tab features (red). Not all the lines overlap though since the starting potentials chosen for the contour lines were different in this case (this is ok). If you like, you can enable Data Recording options (Particles|Data Recording) to record certain parameters on every time-step. For example recording x, y, and z will output the points on the contour lines, and you could, for example, have this saved to a text file (delimited format) and loaded into Excel.
  • 5. Plotting a field generated from a user program... Now let's explore plotting a magnetic field generated from a user program, such as the magnetic field computed from an air core solenoid coil of wire using the simionx.MField library. Load solenoid.iob. (From the Workbench tab click Load... and select solenoid.iob, or go back to the main screen, click Remove All PAs from RAM, click View/Load Workebench and select this file.) Fly the ions. (Click Fly'm.) A number of field lines are plotted. Note that the particle mass is set to 1003 (magnetic field lines). Note also the particle definitions: particles are started along the x-axis with both positive (+) and negative (-) particles at each point to trace the field lines in both directions starting at that point.
  • 6. For further details, examine the code and comments in the Lua files: drag.lua, solenoid.lua, and contourlib.lua.
